Changelog

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog and this project adheres to Semantic Versioning.

[UNRELEASED] - 2017-11-20

Added

  • add ability to customise the UserToolbarComponent template.
  • add the ability to add custom classes to the contents of the UserToolbarComponent
  • add ability to show the start time/date for cart items in the CartComponent. The format can be defined in the CartOptions

Changed

  • update the options for ProductDate and TicketDate components

[v.0.1.12] - 2017-10-20

Added

  • add the ability to customise the UserCompmonent template
  • add the ability to customise the UserLoginComponent within the UserComponent
  • add the ability to select the language and age group for tours using the ProductFiltersComponent

Changed

  • updated the ProductBreadcrumbsComponent to include the language and age group breadcrumbs
  • change the ProductComponent to listen for change events (goComponentChanges) on the element itself, not on the global document level. This allows the support of updating multiple ProductComponents on one page.

Fixed

  • fix calculation of group tour prices in the CartComponent

[v.0.1.11] - 2017-10-17

Added

  • add the dynamic updating of the ProductComponent using a JS CustomEvent
  • allow the use of select fields as the custom template for the ProductTimeComponent
  • add validation to CartActionDirective: only if there are items present in the cart, will the action be carried out
  • added options to disable days for the ProductDateComponent. Dates can now be disabled when they are booked-out and/or empty: IsDateDisabled

Changed

  • Change and unify wording of options input types in documentation
  • changed the user documentation to warn about notifying PMs of the url to be used as the redirect after a successful user registration

Fixed

  • fix the setting of the locale in the global options (_gomusOptions)
  • redirect to the user_url defined in the options object if user is not authenticated in the checkout process

[v.0.1.10] - 2017-10-05

Added

  • add the customisation of user component templates. Now the UserLoginComponent, the UserRegistrationCompmonent and the UserGuestComponent can be fully customised, and the UserComponent partially.
  • Apply classes to the day buttons of the product calender used in the ProductDateComponent. go-empty: for days without any dates or start times. go-booked-out: for days that have no seats or capacity left

Fixed

  • fix missing provider for ProductList and ProductDetail components

[v.0.1.9] - 2017-09-19

Changed

  • Initiate the quantity of group prices for products to 0 not 1
  • Change the ticket's default template option from default_template to default_templates to keep it inline with the products component option

Fixed

  • Fix the calculation of total line price of product prices when a custom template is used

[v.0.1.8] - 2017-08-23

Added

  • add shop_domain attribute to GomusOptions to inform go~mus what shop it is communicating with
  • add action options for the cart button. Supports none, and url.
  • Link demo component pages to the relevant component page in the documentation
  • update documentation to notify reader about providing the url of the checkout and user page for the go~mus configuration
  • emit a gomus-components:loaded event when components have initialised
  • start a best practices section on the README.md page

[v.0.1.7] - 2017-08-04

Added

  • add TicketsDateContainerDirective, TicketsTimeContainerDirective and TicketsTypeContainerDirective to allow better positioning of components within other elements
  • add ProductDateContainerDirective, ProductTimeContainerDirective and ProductPricesContainerDirective to allow better positioning of components within other elements

Fixed

  • trigger step1 when product breadcrumbs are initialised
  • trigger step1 when ticket breadcrumbs are initialised
  • fix showing of event and tour cart items with scale prices

[v.0.1.6] - 2017-08-01

Added

  • add initial inline documentation for components
  • add automatic changelog for documentation
