go~mus components

This is the next version of the go~mus components and is currently under rapid development. To get used to the concepts and try things out, head over to the stable version.

What is go~mus components?

go~mus is a web-based software for visitor management in museums and cultural institutions. It provides the means of selling tickets or products online through different add-ons such as through an online shop or using the API.

These cover both extremes of the spectrum:

  • The online shop allows you to hit the ground running and start selling as soon as possible, but has often the disadvantage of double content management and navigating your users away from your website.
  • The API allows you to build a fully integrated application to whatever system you are currently using, but has the disadvantage of taking more time and resources to set up.

go~mus components is a project that aims to meet you in the middle. It abstracts away the complicated logic that needs to happen in order to sell tickets and products from go~mus, and provides you a simple HTML API to use instead while still allowing a lot of customisation.

Simply put, it allows you to embed small widget-like components into your existing website to enable the selling of tickets, events and tours from go~mus.

Assumptions

This documentation assumes that you are already familiar with HTML and have a basic understanding of JavaScript and JSON. The code samples are written in HTML5, JSON and the latest JavaScript.

Compatibility

This project is compatible with version 4 of the go~mus API. The documentation for which can be found online under: https://giantmonkey.github.io/gomus-api-doc

For browser compatibility please see the browser support section under setup and deployment.

Project status

go~mus components is currently in beta and under active development. During beta, new features will be added regularly and APIs will evolve based on user feedback.

IMPORTANT: Custom tables for dynamic component parts are not possible due to strict browser rules. Flex or other CSS will need to be used instead.
