Setup and Deployment

The setup for you test environments and you production environments should vary to enable the best experience for you and your development environment and to your end users.

You should always load the components script from our CDN in an asynchronous manner. For example:

<script src="https://cdn.gomus.de/components/latest/components.min.js" async></script>

Setup for local and staging environments

It is important that you set the domain for the go~mus instance to a staging environment. Clients normally have two environments, a staging and production environment, and for development the staging environment should always be used.

Staging environments can be identified by the -staging part of the domain. For example if you have a go~mus instance called my, then the production url would be my.gomus.de and the staging instance my-staging.gomus.de.

It is recommended that you develop against the latest version of the components using either /latest/components.min.js or /v.0/components.min.js. This means you keep up to date with the latest changes and bug fixes.

It is also recommended that you set the debug property in the global configuration to true. This enables the outputting of hints, warnings and errors in the browser console.

Setup for production environments

Before going live you should check that you global configuration is correct.

  • The domain needs to be pointed to the production instance of go~mus.
  • The shop_domain needs to be set to your production website URL so that the redirects work properly.
  • The correct payment mode ids need to entered
  • Debug mode should not be disabled by setting it to false, or removing it from the config completely.
  • It is recommended to use the pin the components to a specific version, or minor version if you want to receive patch updates. For example /v.1.2.3/components.min.js or /v.1.2/components.min.js.

Browser support

go~mus components supports the most recent two versions of all major browsers: Chrome (including Android), Firefox, Safari (including iOS), and IE11 / Edge.
