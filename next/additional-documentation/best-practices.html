Best Practices

  • If your page contains a lot of JavaScript, load the component script at end of body or make sure your page has finished rendering before loading the components.
  • Hide ng-template elements using css
    ng-template {
display: none;
}
  • Provide a loading state for components until the components script has finished loading. A gomus-components:loaded event is fired on the document when the initialisation is completed. For example:
    document.addEventListener('gomus-components:loaded', function () {
 console.log('loaded');
});
  • Custom tables for dynamic component parts are not possible due to strict browser rules. Flex or other css will need to be used instead.
  • Make sure you have defined the global configuration to communicate with the correct go~mus instance. There are normally two for each client, a staging/test instance and a production instance. For development make sure you are using the staging environment. If you are unsure then contact your project manager.
