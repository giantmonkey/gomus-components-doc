Getting Started

Getting started is easy and simple. All that is needed is to load the go~mus component JavaScript file and add some components to your HTML page.

By the end of this guide you'll have a basic understanding of how to develop with the components and how to integrate them into your website.

Step 1. Create a basic HTML page

Create a HTML page with the following basic structure:

<!doctype html>

<html lang="en">
  <head>
    <meta charset="utf-8">

    <title>My first go~mus components website</title>
    <meta name="description" content="My first go~mus components website">
    <meta name="author" content="go~mus">

    <!--[if lt IE 9]>
      <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.js"></script>
    <![endif]-->
  </head>

  <body>
    <h1>My first go~mus components website</h1>
  </body>
</html>

Step 2. Load the components script

Next we need to load the components script into the page. To do this will simply need to define a script tag pointing to the location of the components script on the go~mus CDN. The script tag can look like this:

<script src="https://cdn.gomus.de/components/latest/components.min.js" async></script>

We can insert this at the end of the head section so that our HTML page now looks like the following:

<!doctype html>

<html lang="en">
  <head>
    <meta charset="utf-8">

    <title>My first go~mus components website</title>
    <meta name="description" content="My first go~mus components website">
    <meta name="author" content="go~mus">

    <!--[if lt IE 9]>
      <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.js"></script>
    <![endif]-->

    <script src="https://cdn.gomus.de/components/latest/components.min.js" async></script>
  </head>

  <body>
    <h1>My first go~mus components website</h1>
  </body>
</html>

Now when we load the page, the component script loaded in an asynchronous manner.

Step 3. Add a component

We are now ready to add our first component. For this example we will add the tickets component.

To do this we need to add the HTML snippet for the tickets component to the body of our HTML page.

The snippet looks like this:

<go-tickets></go-tickets>

And when added to our HTML page it should look like the following:

<!doctype html>

<html lang="en">
  <head>
    <meta charset="utf-8">

    <title>My first go~mus components website</title>
    <meta name="description" content="My first go~mus components website">
    <meta name="author" content="go~mus">

    <!--[if lt IE 9]>
      <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.js"></script>
    <![endif]-->

    <script src="https://cdn.gomus.de/components/latest/components.min.js" async></script>
  </head>

  <body>
    <h1>My first go~mus components website</h1>
    <h2>My tickets component</h2>
    <go-tickets></go-tickets>
  </body>
</html>

This time when we load the page, the component script loads and sets up the tickets component using the default settings and template.

Hooray, we have our first component! We are now able to select a desired ticket and add it to the cart.

Step 4. Customise the component

We can now customise the component by providing an options JSON Object.

Let's say we want to only display day tickets, instead of all available tickets. In this case we would provide an options object with the new types property.

{
  "types": ["day"]
}

We would then pass the object into the tickets component like this:

<go-tickets options='{"types": ["day"]}'></go-tickets>

Now when we reload our page, only day tickets will be displayed by the component.

Step 5. Define the global configuration

Up until now we have been using the default configuration for the components. To customise further we can define a global variable called _gomusOptions and pass in our own configuration.

We use this to define global options such as what go~mus intance we want the components to communicate with.

The default configuration that we have been using communicates to our demo go~mus instance. To change the go~mus instance to communicate with another instance we would define the following:

var _gomusOptions = {
  domain: "my.gomus.de"
}

We can add this to our HTML page by defining it in another script tag before we load the components file.

<!doctype html>

<html lang="en">
  <head>
    <meta charset="utf-8">

    <title>My first go~mus components website</title>
    <meta name="description" content="My first go~mus components website">
    <meta name="author" content="go~mus">

    <!--[if lt IE 9]>
      <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.js"></script>
    <![endif]-->

    <script type="text/javascript">
      var _gomusOptions = {
        domain: "my.gomus.de"
      };
    </script>
    <script src="https://cdn.gomus.de/components/latest/components.min.js" async></script>
  </head>

  <body>
    <h1>My first go~mus components website</h1>
    <h2>My tickets component</h2>
    <go-tickets></go-tickets>
  </body>
</html>

Obviously my.gomus.de doesn't exist so this will not work. To get the domain name for the instance you should work with, please contact your project manager.

NOTE: The config variable needs to load first before the component script loads.

NOTE: Detailed documentation for this variable can be found under the GomusOptions class.
